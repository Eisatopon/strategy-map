1. Extremal Element

Where to start â€” the most constrained object

In many mathematical problems, the main difficulty is not computation, but deciding where to start.

The Extremal Element strategy is based on a simple principle: start where freedom is minimal.

When to use:
  • "At most" / "at least" statements
  • Minimum or maximum conditions
  • Uniqueness arguments
Thinking steps:
  1. Identify the extremal element
  2. List what it cannot do
  3. See how it affects others
  4. Use for conclusion or contradiction

ğŸ“˜ Basic Level Problem

Given a finite set of positive integers where all pairwise sums are distinct, prove it contains at most one even number.

Hint: What happens if there are two even numbers? Consider the extremal element: the smallest even number. Can you find two pairs with the same sum?

ğŸ“™ Intermediate Level Problem

Given \(n\) points in the plane (not all collinear), prove there exists a point such that, if removed, the minimal distance between remaining points doesn't decrease.

Hint: What is the minimal distance among all points? There's a pair achieving it. What happens if you remove a point not in this pair?

ğŸ“• Advanced Level Problem

In a connected graph with \(n\) vertices, prove there exists a vertex whose removal doesn't increase the number of connected components by more than 1.

Hint: Consider a spanning tree of the graph. Which vertex is the extremal element? (Hint: a leaf has only one edge!)

4. Invariants

Find what stays constant in the chaos of change

Imagine a system that constantly changes: numbers are replaced, positions shift, states transform. In all this chaos, is there something that never changes?

The Invariants strategy is based on a powerful idea: find a quantity that is preserved in every allowed step. If initial and final states have different values for this quantity, then the transition is impossible.

When to choose it:
  • When the problem describes a process or allowed steps
  • When asking "is it possible to..."
  • When there are mod, parity (even/odd), or colorings
  • When something "seems" to stay constant
How to think (step-by-step):
  1. Identify a quantity that might stay constant
  2. Check it is preserved in every allowed step
  3. Compare initial and final states
  4. Use the difference to prove impossibility

ğŸ“˜ Basic Level Problem

Numbers \(1, 2, 3, \ldots, n\) (with \(n \geq 2\)) are written on a circle. You may replace two adjacent numbers with their sum. Is it possible to reach a state where all numbers are equal?

Consider the parity (even/odd) of numbers. Initially there's at least one odd number. When replacing two adjacent numbers with their sum, how does the count of odd numbers change? (even+even=even, even+odd=odd, odd+odd=even)
Solution: The parity of the count of odd numbers is invariant. Initially â‰¥1 odd exists (odd count). If all equal k: either all even (0 odd) or all odd (n odd). If n is even, final count is even - contradiction! âœ“

ğŸ“™ Intermediate Level Problem

3 boxes with balls. You may double one box by taking from the other two. Can you reach all 3 boxes having equal balls?

Let boxes have \(a,b,c\) balls. When doubling the first: new state is \((2a, b-a, c-a)\). Check the differences modulo 3: \(a-b, b-c, c-a\) before and after.
Solution: Differences mod 3 are invariant. If final state is \((k,k,k)\), all differences are 0 mod 3. So only possible if initially \(a \equiv b \equiv c \pmod{3}\). Otherwise impossible! âœ“

ğŸ“• Advanced Level Problem

An \(8 \times 8\) board has integers. You may add 1 to all squares in a row or column. If initial sum of all numbers is not divisible by 8, prove you cannot reach all squares having the same number.

Let \(S\) = sum of all 64 numbers. When adding 1 to a row: 8 squares increase â†’ \(S\) increases by 8. What happens to \(S \bmod 8\)? If all squares equal \(k\): \(S = 64k = 8(8k)\).
Solution: \(S \bmod 8\) is invariant (adding 1 to row/column adds 8). Initial: \(S_0 \not\equiv 0 \pmod{8}\). Final (all equal k): \(S_f = 64k \equiv 0 \pmod{8}\). Contradiction! âœ“

  • Ignoring initial: Only checking final state, forgetting initial
